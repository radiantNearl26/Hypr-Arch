**Step 1: Create a bare local repository**
	⇨ *Initialize a bare Git repository in a new directory (e.g., `~/.dotfiles`)*

```
git init --bare ~/.dotfiles
```

**Step 2: Add an alias for convenience**
	⇨ *Create an alias to run Git with specific options*

```
alias dotfiles='/usr/bin/git --git-dir="$HOME/.dotfiles/" --work-tree="$HOME"'
```

**Step 3: Set status to show untracked files to no**
	⇨ *Set the `status.showUntrackedFiles` option to `no` to avoid cluttering the `dotfiles status` output*

```
dotfiles config status.showUntrackedFiles no
```

**Step 4: Add and commit demo files**
	⇨ *Create some demo files in your home directory (e.g., `~/.bashrc`, `~/.vimrc`)*
	⇨ *Add these files to the Git repository*

```
dotfiles add <file>
```
*	⇨ Commit the changes*
```
dotfiles commit -m "Initial commit"
```

**Step 5: Set up a link to a remote repository using SSH URL**
	⇨ *Create a remote repository on GitLab (e.g., `arch-dotfiles`)*
	⇨ *Add the remote repository to your local repository using the SSH URL*
```
dotfiles remote add origin git@gitlab.com:your-username/arch-dotfiles.git
```
*	⇨ Push your local changes to the remote repository*
```
dotfiles push -u origin master
```

***That's it! Follow these steps to set up your dotfiles repository.***
